# Spring Boot sample app (intentionally vulnerable, multi-module)

A small, **real** multi-module (reactor) Spring Boot Maven app used as the end-to-end test
bed for `dep-remediation`. Its poms are committed in a deliberately **vulnerable "before"
state** so the sample is reusable вЂ” runs of the tool should be made against a copy.

> вљ пёЏ This app pins **old, vulnerable** dependency versions on purpose. Do not use it as a
> starting template for real work.

## Layout

```
spring-boot-sample/        aggregator (packaging pom) вЂ” managed/transitive security pins go here
в”њв”Ђв”Ђ sample-core/           library module: the direct + property findings
в””в”Ђв”Ђ sample-web/            Spring Boot app module: the managed + transitive findings
```

`sample-web` depends on `sample-core`, so several findings resolve in **both** modules вЂ” the
reactor-wide resolution check (`verify` at the root) covers every module.

## Seeded findings вЂ” one per resolution class

| Library | Where / how | Resolution class | Before в†’ recommended |
|---------|-------------|------------------|----------------------|
| `org.apache.commons:commons-text` | `sample-core`, literal `<version>` | **direct** | 1.9 в†’ 1.10.0 |
| `org.apache.commons:commons-collections4` | `sample-core`, `${commons-collections4.version}` | **property** | 4.1 в†’ 4.4 |
| `org.yaml:snakeyaml` | `sample-web`, versionless (Boot parent BOM) | **managed** | 2.2 в†’ 2.3 |
| `org.apache.tomcat.embed:tomcat-embed-core` | `sample-web`, via `spring-boot-starter-web` | **transitive** | 10.1.24 в†’ 10.1.25 |

The matching advisory is `advisory.xlsx`, regenerated by `make_advisory.py` (it also seeds
noise rows вЂ” a different owner, a container/OS row, and a base-image row вЂ” to exercise the
parser's filter chain).

## Fix targeting (automatic)

A `<dependencyManagement>` pin only affects its own module subtree and **cannot override a
module's explicit `<version>`**, so each finding must land in the right pom. The engine does
this **automatically** вЂ” point it at the project and it routes:

- **direct / property** (`commons-text`, `commons-collections4`) в†’ edited in **`sample-core/pom.xml`**.
- **managed / transitive** (`snakeyaml`, `tomcat-embed-core`) в†’ pinned in the **aggregator
  `pom.xml`** so every module inherits it.

## Try it end to end

```bash
# from the repo root
python examples/spring-boot-sample/make_advisory.py   # (re)generate advisory.xlsx + advisory-breaking.xlsx

# work on a COPY so the committed sample stays vulnerable
cp -r examples/spring-boot-sample /tmp/sample && cd /tmp/sample

# one command fixes the whole reactor (auto-targeted across modules)
dep-remediation fix . --from-advisory advisory.xlsx --app sample-app --apply
# verify the whole reactor at the aggregator root
dep-remediation verify . --from-advisory advisory.xlsx --app sample-app
```

A captured happy-path run lives in [`SHAKEOUT.md`](SHAKEOUT.md). `make_advisory.py` also
emits `advisory-breaking.xlsx` (one finding recommends a non-existent version) to drive the
**build-failure recovery loop** вЂ” a captured recovery run is in [`RECOVERY.md`](RECOVERY.md).

## Requirements

- JDK 17+ and Maven (a project `mvnw`/`mvnw.cmd` wrapper is included at the root; system
  `mvn` also works). The committed "before" state builds green (vulnerable, but compiles):

```bash
cd examples/spring-boot-sample && ./mvnw clean install   # or: mvn clean install
```
