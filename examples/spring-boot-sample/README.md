# Spring Boot sample app (intentionally vulnerable)

A small, **real** single-module Spring Boot Maven app used as the end-to-end test bed for
`dep-remediation`. Its `pom.xml` is committed in a deliberately **vulnerable "before"
state** so the sample is reusable — runs of the tool should be made against a copy.

> ⚠️ This app pins **old, vulnerable** dependency versions on purpose. Do not use it as a
> starting template for real work.

## Seeded findings — one per resolution class

| Library | In pom as | Resolution class | Before → recommended |
|---------|-----------|------------------|----------------------|
| `org.apache.commons:commons-text` | literal `<version>` | **direct** | 1.9 → 1.10.0 |
| `org.apache.commons:commons-collections4` | `${commons-collections4.version}` | **property** | 4.1 → 4.4 |
| `org.yaml:snakeyaml` | versionless (Boot parent BOM) | **managed** | 2.2 → 2.3 |
| `org.apache.tomcat.embed:tomcat-embed-core` | not declared (via `spring-boot-starter-web`) | **transitive** | 10.1.24 → 10.1.25 |

The matching advisory is `advisory.xlsx`, regenerated by `make_advisory.py` (it also seeds
noise rows — a different owner, a container/OS row, and a base-image row — to exercise the
parser's filter chain).

## Try it end to end

```bash
# from the repo root
python examples/spring-boot-sample/make_advisory.py          # (re)generate advisory.xlsx

dep-remediation parse examples/spring-boot-sample/advisory.xlsx --app sample-app
dep-remediation fix   examples/spring-boot-sample/pom.xml \
    --from-advisory examples/spring-boot-sample/advisory.xlsx --app sample-app      # dry-run diff

# apply + verify — do this on a COPY so the committed sample stays vulnerable
cp -r examples/spring-boot-sample /tmp/sample && cd /tmp/sample
dep-remediation fix pom.xml --from-advisory advisory.xlsx --app sample-app --apply
dep-remediation verify . --from-advisory advisory.xlsx --app sample-app
```

A captured happy-path run lives in [`SHAKEOUT.md`](SHAKEOUT.md). `make_advisory.py` also
emits `advisory-breaking.xlsx` (one finding recommends a non-existent version) to drive the
**build-failure recovery loop** — a captured recovery run is in [`RECOVERY.md`](RECOVERY.md).

## Requirements

- JDK 17+ and Maven (a project `mvnw`/`mvnw.cmd` wrapper is included; system `mvn` also
  works). The committed "before" state builds green (vulnerable, but compiles):

```bash
cd examples/spring-boot-sample && ./mvnw clean install   # or: mvn clean install
```
